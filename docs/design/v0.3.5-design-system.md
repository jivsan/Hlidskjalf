# Hlidskjalf v0.3.5 — design system (apply this, don't reinvent it)

The visual language is already built and LOCKED in these files (read them, do not
edit them): `tailwind.config.js`, `src/index.css`, `src/main.tsx`,
`src/components/ui.tsx`, `src/components/Layout.tsx`, `src/pages/Login.tsx`,
`src/components/charts.tsx`. Your job is to apply that language to page files so
every screen shares one rhythm.

## Thesis
Hlidskjalf is the operator's high seat, watching every guest (VM) running on the
host "pve". **Machine data is set in monospace (JetBrains Mono via `.metric`),
the human interface in Archivo (the default `font-sans`/`font-display`).** That
split is the concept — honor it: numbers, IDs, UPIDs, MACs, IPs, byte counts →
wrap in `.metric` (or a `font-mono` container). Labels, headings, prose → default.

## Color tokens (Tailwind) — use ONLY these, never a raw hex
- `bg` deep night · `abyss` deepest (data wells) · `surface` card · `surface-2` raised (active rows/menus) · `border-token` hairlines
- `fg` text · `muted` secondary text
- `cyan` = live / healthy / primary signal · `pink` = brand + selection/active ONLY · `amber` = attention/warning · `red` = danger
Discipline: **do not spray pink.** Most accents should be cyan (live data) or muted. Pink marks the brand and the current selection, nothing else.

## Shared components (import from `../components/ui` or `../../components/ui`)
- `<PageHeader eyebrow title sub? actions? />` — **every page must open with this.** `eyebrow` is a short lowercase section label, `title` is the plain name in the display face.
- `<Card title? actions?>` — title renders as a small uppercase eyebrow label automatically.
- `.well` class — recessed dark panel for data (logs, config dumps, code).
- `.eyebrow` class — labeled hairline (uppercase, tracked). Used by PageHeader; reuse for sub-sections.
- `<LoadingState/>`, `<EmptyState/>`, `<ErrorState/>`, `<StatusDot/>`, `<ProgressBar/>`, `<KindBadge/>` — already restyled; use as-is.
- Buttons: `.btn-cyan .btn-pink .btn-plain .btn-amber .btn-red`. Inputs: `.input`. Labels: `.label`.

## Required PageHeader eyebrows (exact copy — plain + useful, no theme-forcing)
- Fleet → eyebrow `guests on <node>` (the live node name), title `Fleet`
- Node → eyebrow `hypervisor host`, title the node name (e.g. `pve`)
- Provision → eyebrow `new guest`, title `Provision`
- Users → eyebrow `access control`, title `Users`
- Debug → eyebrow `diagnostics`, title `Debug`
- VM detail → eyebrow `guest`, title the VM name

## Hard rules
1. **Do not change behavior, props, handlers, data flow, or API calls.** Visual + markup only.
2. **No new colors, hex values, fonts, or `@font-face`.** No inline `style={{color/background}}` with raw hex (chart color constants from `charts.tsx` are fine to pass through).
3. **No new animations** beyond what exists (`.reveal`, `.spinner`, StatusDot ping). Reduced-motion is handled globally — don't fight it.
4. Keep it responsive (mobile → desktop) and keep visible keyboard focus (`:focus-visible` is global; don't remove outlines).
5. Only edit the page files assigned to you. Leave every LOCKED file untouched.
6. Run `npx tsc --noEmit` in `frontend/` before finishing; it must pass.

## Taste
Match complexity to a working operator console: precise, dense, quiet. Spend no
boldness here — the masthead and login already carry the signature. Your pages
should feel calm and consistent: consistent card padding, consistent eyebrow
rhythm, data in mono, generous but not loose spacing. Remove decoration that
doesn't inform. When unsure, make it quieter.
